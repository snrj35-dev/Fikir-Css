# v0.5.0 Release Notes

## Summary

v0.5.0 promotes the input augmentation slice (combobox, search-box, autocomplete, command-palette) to `supported`, ships a publishable npm package under the `fikir-css` name, adds opt-in theme distribution, and fixes all known package surface and doc-contract drift issues.

## Highlights

- **combobox / search-box / autocomplete / command-palette** promoted from experimental to supported (M2 Wave 6)
- Package renamed from `fikir-css-mvp` to **`fikir-css`** — publishable to npm
- `dist/tooling/resolve-classes.mjs` now ships in the tarball; `fikir-css/tooling` and `fikir-css/resolvers` imports work out of the box
- `dist/themes/` added to tarball: `dark`, `light`, `high-contrast`, `compact`, `comfortable`, `reduced-motion`, `shape`
- Gzip budget CI guardrail made explicit (separate step in `ci.yml`)
- Screenshot CI added: `capture:playground` + `screenshot:diff` (non-blocking, `continue-on-error: true`)
- Canonical selector drift fixed across all adapter docs and migration guides

## Behavioral Notes

- Close buttons in modal examples now use `icon-button` (canonical) instead of non-existent `modal-close` class
- Tab panels now carry `role="tabpanel"` in playground HTML
- `tabs-trigger` / `tabs-panel` are the canonical class names (not `tab-trigger` / `tab-panel`)

## Upgrade Notes

- v0.4.0 → v0.5.0
- Rename `fikir-css-mvp` → `fikir-css` in your `package.json` dependency
- Replace `modal-panel` → `modal-dialog`, `modal-close` → `icon-button`, `tab-trigger` → `tabs-trigger`, `tab-panel` → `tabs-panel`, `btn--sm` → `btn-sm`
- Theme CSS files moved from `packages/tokens/themes/` (source) to `dist/themes/` (published); import via `fikir-css/themes/dark` etc.
