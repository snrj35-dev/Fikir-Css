# Fikir CSS v0.4.0 Release Notes (Draft)

> Last reviewed: 2026-04-11
> Status: draft (not published)

## Summary
v0.4.0 focuses on professionalization: support-level promotions, workflow-complete playground examples, and stronger visual/a11y quality gates.

## Milestone Scope
Completed in this draft scope:
- Support promotions: `textarea`, `select`, `checkbox`, `radio`, `switch`, `toast`, `pagination`
- Gap components: `number-input`, `rating`, `result`, `tree-view`, `tags-input`
- Workflow examples: settings-step validation, tree+table workflow, result/status outcomes
- Visual QA: deterministic capture command and supported-slice baseline set

Deferred:
- External feedback volume goals and full pixel-diff CI
- GitHub project automation actions requiring authenticated external access

## Highlights
1. Support-level promotions (experimental -> supported)
- Forms baseline promoted: `textarea`, `select`, `checkbox`, `radio`, `switch`
- Overlay baseline expanded: `toast`
- Navigation/data baseline expanded: `pagination`
- Matrix update: `docs/roadmap/support-matrix.md`

2. Workflow-level documentation examples
- `playground/settings-workflow-example.html`
- `playground/tree-table-workflow-example.html`
- `playground/result-status-example.html`

3. Visual regression baseline progression
- Command: `npm run capture:playground`
- Baseline validator: `npm run validate:playground-baseline`
- Supported-slice snapshots: `playground/screenshots/sections/*.png`

## Behavioral Notes
- No selector removals are introduced for promoted surfaces.
- Support-level intent changed for listed surfaces; they are now part of the committed baseline.
- Playground now includes dedicated workflow/result/toast pages used for visual baselines.

## Upgrade Notes
- Consumers can treat promoted surfaces as supportable baseline in v0.4.
- If your process tracks support-level commitments, update your internal mapping from `experimental` to `supported` for the promoted surfaces.
- Continue to run quality gates before release publication:
  - `npm run test:ci`
  - `npm run validate:publish`
  - `npm run package:smoke`
