# v0.6.0 → v1.0.0 Migration Note

> Source: `v0.6.0`
> Target: `v1.0.0`
> Channel: `latest`
> Published: 2026-04-19

## Summary

v1.0.0 is a **non-breaking stable release** for all `supported` surfaces. No selector renames, token renames, or behavioral changes were made to the 69 supported surfaces since v0.6.0.

The primary additions are: machine-readable JSON contracts (M20), framework example apps (M18), CI hardening (M19), and the support-level freeze declaration (M21.1).

---

## Migration Impact

**None for supported surfaces.** If you use only the 69 supported surfaces, you can upgrade with no code changes:

```bash
npm install fikir-css@1.0.0
```

---

## What changed in 1.0.0

### New: Machine-readable contracts (M20)

Six JSON manifests are now published in `dist/contracts/` and available via npm subpath imports:

```js
import anatomy      from "fikir-css/contracts/anatomy";
import capabilities from "fikir-css/contracts/capabilities";
import variants     from "fikir-css/contracts/variants";
import primitives   from "fikir-css/contracts/primitives";
```

These are additive — no existing import paths changed.

### New: Framework example apps (M18)

Runnable starter apps added under `examples/`:
- `examples/react-vite` — React 18 + Vite
- `examples/vue-vite` — Vue 3 + Vite
- `examples/svelte-vite` — SvelteKit

### Support level freeze (M21.1)

The `supported` surface list is now frozen under semver:
- **Breaking changes** to any supported surface require a **MAJOR** version bump.
- **Beta** surfaces may receive additive changes in **MINOR** releases (documented in CHANGELOG).
- **Experimental** surfaces carry no semver guarantee.

See `docs/roadmap/support-matrix.md` for the full surface classification.

---

## Supported Surface Freeze Check

All 69 supported surfaces are unchanged from v0.6.0. No selector was renamed, removed, or had its token bindings changed.

Freeze is declared in `docs/roadmap/support-matrix.md`:
> **M21.1 FREEZE:** The `supported` list is final. No new surfaces will be added to `supported` without a MAJOR version bump.

## Selector / Alias Changes

**None.** No selector renames were made between v0.6.0 and v1.0.0.

`dist/contracts/alias-migration.json` covers v0.x → v1.0 alias mappings if you are upgrading from v0.2–v0.5.

---

## Token / Behavior Changes

No token values changed for supported surfaces. The following were additive:
- `dist/contracts/tokens.json` now includes 59 tokens across 7 groups with px equivalents.
- `data-*` attribute documentation added to `selectors.json`.

## Beta Surface Changes

The following surfaces moved from "not yet implemented" (M16 roadmap) to **beta** status:

| Surface | Notes |
|---------|-------|
| `time-picker` | Format/step contract not yet frozen |
| `date-time-picker` | Combined date+time contract; locale not frozen |
| `inline-notice` | Semantic distinction from alert/callout pending |
| `command-bar` | Slot contract may evolve |
| `tree-table` | Row expand/collapse keyboard contract pending |
| `copy-button` | Clipboard API pattern may be extended |
| `password-input` | Reveal/hide toggle state pending |
| `stat-group` | Responsive contract not frozen |
| `empty-search-state` | Pattern-based (`data-pattern`); variant contract may grow |

The following moved to **experimental**:

| Surface | Notes |
|---------|-------|
| `coachmark` | Tour sequencing API not defined |
| `auth-screen` | Opinionated layout; slot contract not frozen |

---

## Release Channel Guidance

- **Stable production use:** Upgrade to `v1.0.0` immediately — no code changes required for supported surfaces.
- **Beta surfaces:** Continue using with caution; minor additive changes possible in `v1.x`.
- **Experimental surfaces:** No semver guarantee. Pin to exact version if using in production.

---

## Recommended Migration Steps

1. Update `package.json`:
   ```bash
   npm install fikir-css@1.0.0
   ```
2. Run `npm run build` — manifests will be regenerated automatically.
3. Optionally import new machine-readable contracts for AI/tooling integration:
   ```js
   import anatomy from "fikir-css/contracts/anatomy";
   ```
4. Review `docs/roadmap/support-matrix.md` for updated surface classifications.
5. No selector or token changes needed in application code.
