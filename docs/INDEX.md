## Support-level Notes
- `supported`: committed in current support baseline (`docs/roadmap/support-matrix.md`)
- `experimental`: implemented and demoed, but outside supported commitment
- `rfc-only`: spec exists without canonical implemented surface
- `planned`: deferred roadmap scope

## Core Docs Metadata Convention
- Convention doc: `docs/maintainer/last-reviewed-convention.md`
- Core docs should include `Last reviewed: YYYY-MM-DD` under the title.
- Use `Status: stale` only for historical documents that should not drive current decisions.

## Status Banner Convention
- `Status: stale` means historical context only; prefer active roadmap/release docs for current decisions.
